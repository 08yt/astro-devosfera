## 📺 为什么要在 Apple TV 上原生部署 Sing-box？

以往为了让客厅的电视盒子科学上网，大家通常有以下几种折腾手段：

1. **软路由/旁路由**：全家网关统一代理，技术门槛高，设备耗电且偶尔会拖累非翻墙设备的网速。
2. **局域网代理共享**：利用电脑或手机的小火箭/Clash 进行局域网共享代理，每次看电视必须保持电脑开机，极度不优雅。

而有了 tvOS 原生 **SFI（Sing-box for iOS/tvOS）** 之后：

- **零外接硬件**：只需一台 Apple TV 盒子即可自主接管其所有的网络流量。
- **苹果原生架构支持**：基于 SwiftUI 开发，深度适配 Apple TV 遥控器的手势操作与后台运行逻辑，息屏不掉线，低功耗且稳定。
- **终极性能释放**：Sing-box 底层内核的极佳并发性能，能吃满你的宽带物理上限，完美解决高码率 4K/8K 视频播放时的缓冲卡顿。

## 📥 第二步：导入订阅节点（两种黄金方案）

安装完成后，打开 Sing-box 客户端（以下简称 SFI）。在电视屏幕上用遥控器逐字输入长串的订阅链接是极其痛苦的噩梦。因此，Sing-box 提供了非常人性的**跨端同步/网页管理**功能。

### 方案 A：利用 iCloud 云同步导入（最优雅）

由于 SFI 完美支持苹果生态的 **iCloud 备份同步**：

1. 如果你使用的是 iPhone，直接在你的 iPhone 上安装官方 Sing-box 客户端。
2. 在 iPhone 端打开客户端，在 `Profiles` 菜单中新建你的订阅配置（或者通过我们之前分享的 [《Subconverter 订阅转换教程》](/posts/sing-box-subconverter-guide/)，将原始机场配置转化为完美的 Sing-box JSON）。
3. 确保你的 iPhone 和 Apple TV 登录了**同一个 iCloud 账号**，且在 iCloud 设置中开启了 Sing-box 的云同步权限。
4. 几秒钟后，在 Apple TV 端的 Sing-box 客户端中，你就会惊喜地发现刚才导入的配置文件已经自动静默同步在列表中了！

### 方案 B：利用局域网 Web 控制台导入（最直接）

1. 在 Apple TV 端打开 SFI，进入 `Settings`。
2. 找到 `Remote Dashboard` 或 `Local Link` 相关的局域网管理选项，点击启用。
3. 此时屏幕上会显示一个 IP 地址和端口（例如：`http://192.168.1.15:9090`）。
4. 在你同一局域网下的电脑或手机浏览器中，直接输入这个 IP 网址，即可打开 SFI 的局域网网页后台。
5. 在网页端，你可以非常方便地粘贴长串的订阅链接，或者直接将手搓好的 `config.json` 上传到 Apple TV 端。

## ⚙️ 第三步：大屏幕流媒体与 AI 工具智能分流（JSON 进阶）

在 Apple TV 上，我们希望实现最完美的**无感分流**：

1. **国内直连流媒体**（如腾讯视频、爱奇艺或 Bilibili 等国内直连流量）-> **直连出站（Direct）**，确保国内速度且不浪费代理流量。
2. **常规流媒体**（YouTube、Netflix、Disney+、HBO Max）-> **媒体解锁策略组（Media-Group）**，选用延迟最低或原生解锁的地区节点。
3. **AI 智能工具**（OpenAI ChatGPT / Apple Intelligence）-> **纯净 IP 节点（AI-Group）**。

下面是适用于 SFI 的精简路由规则逻辑，您可以将其整合进您的 `config.json` 配置文件中：

```json
{
  "route": {
    "rules": [
      {
        "geosite": ["netflix", "disney", "youtube", "hbo"],
        "outbound": "media-unlock-out"
      },
      {
        "geosite": ["openai", "anthropic"],
        "outbound": "ai-clean-out"
      },
      {
        "geosite": ["cn"],
        "geoip": ["cn", "private"],
        "outbound": "direct"
      }
    ],
    "final": "proxy-auto-fallback"
  }
}
```

在 `outbounds`（出站）设置中：

- `media-unlock-out` 策略组建议使用 `urltest` 特性，绑定数条香港、台湾或日本的高速流媒体节点，它会每隔几分钟自动检测丢包，静默切换至速度最快的节点，保证你看电影中途**零缓冲、零卡顿**。

## ❓ 常见问题解答与踩坑指南 (FAQ)

**Q：为什么开启 Sing-box 后，Apple TV 提示网络断开或者 HomeKit 智能家居异常？**
A：这通常是由于你的 JSON 配置中 DNS 模块或者 `route` 模块没有避开局域网私有 IP 段。请务必确保你的直连规则（`direct`）中包含了 `geoip: ["private"]`，以便放行 HomeKit 及局域网内其他智能家居设备（如 HomePod、NAS）之间的相互通信。

**Q：tvOS 的后台限制严格，Sing-box 会被系统强行杀掉后台吗？**
A：不会。SFI 接入的是 tvOS 官方合规的 Network Extension（网关扩展）接口。只要你开启了连接，即使 Apple TV 进入休眠/息屏状态，只要它没有彻底断电，Sing-box 核心都会在系统底层稳定常驻，做到全天候开机即享。
