## Caddy：自动挡的究极优雅

在 2025 年，我们终于迎来了这款天生为现代互联网设计的 Web 服务器：**Caddy**。

如果说 Nginx 是那种极其复杂、需要你去深入了解离合、油门、转速原理的手动挡赛车。那 Caddy 就是那辆全自动带智能驾驶的新能源汽车。

Caddy 最恐怖、最让人喜爱、最令人不可思议的神级特性就是：**原生全自动 HTTPS。**

是的，你没有听错。当你安装好 Caddy 时，你**不需要**写任何关于 80 跳转 443 的配置，**不需要**配置复杂的 SSL 证书路径，**不需要**设置任何恶心的自动续期计划任务脚本。

你只需要在配置文件（`Caddyfile`）里写上极其不可思议的一行代码：
```caddyfile
example.com {
    reverse_proxy localhost:8080
}
```
**完成了！这就完事了！**

当你运行 Caddy 的那一瞬间，它会自动在后台：
1. 自己去向 Let's Encrypt 或是 ZeroSSL 发起申请。
2. 自动配置好并且进行验证（HTTP 或 TLS-ALPN 验证）。
3. 帮你把证书无缝拉取下来并自动应用到你的网站。
4. 更逆天的是：一旦证书只剩 30 天要过期了，它会在后台全自动帮你无感续期。

## 极简美学：Caddyfile 吊打 nginx.conf

除了自动化 HTTPS，Caddy 的配置文件 `Caddyfile` 也完美体现了 2025 年的“极客极简审美”。
以你要做反向代理为例，Nginx 的配置可能长这样（这已经是我尽量精简过的）：

```nginx
server {
    listen 80;
    server_name example.com;
    return 301 https://$host$request_uri;
}
server {
    listen 443 ssl;
    server_name example.com;
    
    ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;

    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

而达到同样，甚至包含在内开启 HTTP/3、开启压缩等功能，完整的 Caddy 配置只需要：
```caddyfile
example.com {
    encode gzip zstd
    reverse_proxy localhost:8080
}
```
